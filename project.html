Smart Project

Altop Products

Het project van deze minor gaan wij (Tobias, Edin en ik) uitvoeren bij het bedrijf Altop Products. Altop Products maakt o.a. innovaties op het gebied van van bewateringssystemen voor voetbalvelden d.m.v. DrainTalent en allerlei andere lopende projecten en innovaties.

Op dit moment wil Altop haar capaciteit uitbreiden op het gebied van de infrastructuur van DrainTalent. Aan ons de opdracht om te inventariseren waar het systeem op spaak zou lopen als de digitale infrastructuur zwaarder belast zou worden.

Het Draintalent Systeem

Alles wat wij gaan uitvoeren bij Altop heeft betrekking op het Draintalent Systeem. Draintalent wordt gebruikt bij (proffesionele) voetbalclubs om water te ontrekken en toe te voegen ondergronds. Doormiddel van een speciale techniek is het mogelijk om ondergronds water te voegen.

Als extra dienst voor de Draintalent biedt Altop een software product aan in de vorm van een dashboard. Dit dashboard geeft informatie over de Draintalent unit en bied een interactie met de HMI.

Onze opdracht was initieel het opzetten van een DDOS 'aanval' om te testen hoe bestendig het dashboard is tegen dit type aanval. Nadat hier resultaten uit zijn gekomen, hebben we een opzet gemaakt voor hoe optimalisaties te behalen zijn.

Dit project is opgedeeld in 4 kern opdrachten:

EMQX/Telegraf

EMQX en Telegraf werden gebruikt om de EMQTT berichten te vertalen naar een entry in de database. Omdat we nu een alternatieve database hebben (Timescale), veranderd deze infrastructuur. In deze verandering is er in plaats van telegraf nu een NodeJS applicatie gebouwd die alle entries naar de database afvangt. Deze is configureerbaar en in te stellen naar eigen voorkeur.

API en Dashboard

De API wordt gebruikt voor het ophalen van gegevens uit de database en het verstrekken van authorisatie van de gebruiker. Binnen deze applicatie zijn er met name veranderingen gemaakt bij het ophalen van de gegevens. Dit is gedaan om het ophalen van gegevens sneller te maken. Het Dashboard maakt uiteindelijk gebruik van het dashboard. De frontend van het Draintalent Dashboard is geschreven in React. Om de applicatie soepeler te laten werken, zijn er veranderingen aangebracht aan hoe de React Componenten samen werken in het dashboard. Dit heeft voordelen voor onderhoudbaarheid, veiligheid en snelheid.

SLA

Alle deelopdrachten zijn tot nu heel erg gericht op ICT. Om deze opdracht ook wat meer "Bedrijfskunde" te laten zijn, moet er ook een opzet geleverd worden van een SLA. In een SLA staat in ons geval omschreven naar welke eisen software moet voldoen en functioneren.

Database

In database land is heel veel veranderd door ons als team. Orgineel werdt er gebruik gemaakt van InfluxDB als database. Gedurende het testen kwamen we er achter dat deze database engine een flinke bottleneck opleverde. Om deze redene hebben we een verandering naar Timescale gerealiseerd. Hierbij hebben we de InfluxDB geexporteerd, en een custom Rust script geschreven die alle gegevens importeerd in Timescale. Ook zijn deze veranderingen doorgevoerd in de front en back-end.

Eindproduct

SLA en Prototype

Als eindproduct leveren wij Code op die een Prototype maakt. We hebben deze code gerealiseerd naar de resultaten die uit het onderzoek kwamen dat we hebben gedaan naar wat de capaciteit is van het Draintalent Dashboard. Ten tweede leveren wij een SLA (Service Level Agreement) die gebruikt kan worden als opzet voor het Draintalent Systeem. In dit document staat beschreven aan voor eisen het systeem moet voldoen en wat de eindgebruiker kan verwachten van het Draintalent Dashboard

Verslaglegging en Onderzoek

De kern van deze opdracht was een onderzoek naar de capaciteit van het Draintalent systeem. In dit onderzoek staat vermeld waar de pijnpunten liggen. Op basis van dit onderzoek zijn we verder gegaan met het creeren van een prototype.
